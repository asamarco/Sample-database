The sample database is structured as a tree to allow complete flexibility in the types of process flows that can be logged.
Each entry is called a <i>Node</i> and to each Node is associated a <i>Template</i>. Nodes can be freely moved up and down or across the tree hierarchy to represent correctly the flow of operations.
Each template contains specific fields which may be the parameters of a process, images or files, text, tables, graphs...

The example groups samples by location but they can be organized however it seems convenient, templates can be added and improved for specific use cases.

<i>Clone nodes</i> are used in the example with two purposes:

1. A single process acting on multiple samples: node://mky8j5ha-0qkz7trxb
2. A dice, being at the same time a descendant of a sample and a sample per se: node://mky8j5ha-4b187zpmg

Usage of clones in this way is strongly encouraged.
